# GitHub Actions Workflow Reference

## Complete Updated Workflow

This is the complete `.github/workflows/playwright.yml` file with all reporting features.

### Key Sections

#### 1. Permissions Block
Defines the minimum permissions needed for all features:
- `contents: read` — read repository
- `checks: write` — publish test results
- `pull-requests: write` — comment on PRs
- `pages: write` — deploy to GitHub Pages
- `id-token: write` — OIDC authentication for Pages

#### 2. Test Job Steps

**Core steps (unchanged):**
- checkout
- setup-node
- configure git
- verify environment variables
- cache playwright browsers
- install dependencies
- lint
- build
- run tests

**New reporting steps:**
- Parse test results (extracts metrics)
- Create job summary (displays results)
- Upload artifact (backup)
- Publish unit test results (GitHub test interface)
- Comment on PR (sticky comment)
- Upload pages artifact (prepares for deployment)

#### 3. Deploy Job

New `deploy-pages` job that:
- Runs only on `main` branch
- Runs only on `push` events
- Depends on `test` job completing
- Uses official `actions/deploy-pages@v4`
- Deploys to GitHub Pages with environment protection

## Step Explanations

### Parse test results
```yaml
- name: Parse test results
  if: always()
  id: test-results
  run: node scripts/parse-test-results.js > test-summary.json && cat test-summary.json
```
- **When:** Always (even if tests fail)
- **What:** Parses `test-results/results.json` and extracts metrics
- **Output:** `test-summary.json` with passed/failed/skipped/duration
- **Why:** Provides structured data for downstream steps

### Create GitHub Actions Job Summary
```yaml
- name: Create GitHub Actions Job Summary
  if: always()
  run: |
    SUMMARY=$(cat test-summary.json)
    PASSED=$(echo "$SUMMARY" | jq -r '.passed')
    FAILED=$(echo "$SUMMARY" | jq -r '.failed')
    SKIPPED=$(echo "$SUMMARY" | jq -r '.skipped')
    DURATION=$(echo "$SUMMARY" | jq -r '.duration')

    cat >> $GITHUB_STEP_SUMMARY <<EOF
    ## ✅ Playwright Test Results

    | Metric | Value |
    |--------|-------|
    | **Passed** | $PASSED |
    | **Failed** | $FAILED |
    | **Skipped** | $SKIPPED |
    | **Duration** | ${DURATION}s |

    ### 📦 Artifacts
    - **Artifact Name**: \`playwright-report-${{ github.run_id }}\`
    - **Retention**: 30 days

    ### 📄 Reports
    - [View HTML Report (GitHub Actions)](https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }})
    - [GitHub Pages Report](https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}/report-${{ github.run_id }}/)

    EOF
```
- **When:** Always (even if tests fail)
- **What:** Writes formatted summary to `$GITHUB_STEP_SUMMARY`
- **Where:** Appears at top of workflow run
- **Format:** Markdown table with metrics and links

### Publish Unit Test Results
```yaml
- name: Publish Unit Test Results
  if: always()
  uses: EnricoMi/publish-unit-test-result-action@v2
  with:
    files: test-results/junit.xml
    check_name: Playwright Test Results
```
- **When:** Always (even if tests fail)
- **What:** Publishes JUnit results to GitHub
- **Where:** Appears in PR checks and GitHub test interface
- **Format:** Failed tests with stack traces and links
- **Action:** Third-party action (EnricoMi/publish-unit-test-result-action)

### Comment on Pull Request
```yaml
- name: Comment on Pull Request
  if: always() && github.event_name == 'pull_request'
  uses: actions/github-script@v7
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
    script: |
      const fs = require('fs');
      const summary = JSON.parse(fs.readFileSync('test-summary.json', 'utf-8'));

      const body = `## ✅ Playwright Test Report

      **Test Summary:**
      - ✅ Passed: ${summary.passed}
      - ❌ Failed: ${summary.failed}
      - ⏭️ Skipped: ${summary.skipped}
      - ⏱️ Duration: ${summary.duration}s

      **📋 Links:**
      - [HTML Report](https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }})
      - [GitHub Actions Run](https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }})

      ---
      *Generated by Playwright Test Runner*`;

      const { data: comments } = await github.rest.issues.listComments({
        owner: context.repo.owner,
        repo: context.repo.repo,
        issue_number: context.issue.number,
      });

      const botComment = comments.find(comment => 
        comment.user.type === 'Bot' && 
        comment.body.includes('Playwright Test Report')
      );

      if (botComment) {
        await github.rest.issues.updateComment({
          owner: context.repo.owner,
          repo: context.repo.repo,
          comment_id: botComment.id,
          body: body,
        });
      } else {
        await github.rest.issues.createComment({
          owner: context.repo.owner,
          repo: context.repo.repo,
          issue_number: context.issue.number,
          body: body,
        });
      }
```
- **When:** Only on PR events
- **What:** Creates or updates a sticky comment
- **Where:** PR conversation tab
- **Format:** Test summary with emojis and links
- **Smart:** Finds and updates existing comment (no duplicates)
- **Action:** Uses `actions/github-script@v7` for GitHub API access

### Upload Pages Artifact
```yaml
- name: Upload Pages artifact
  if: always() && github.ref == 'refs/heads/main'
  uses: actions/upload-pages-artifact@v3
  with:
    path: playwright-report/
```
- **When:** Always, but only on `main` branch
- **What:** Prepares the HTML report for deployment
- **Format:** Uploads `playwright-report/` directory
- **Next Step:** Consumed by `deploy-pages` job

### Deploy Pages Job
```yaml
deploy-pages:
  if: always() && github.ref == 'refs/heads/main' && github.event_name == 'push'
  needs: test
  runs-on: ubuntu-latest
  environment:
    name: github-pages
    url: ${{ steps.deployment.outputs.page_url }}
  steps:
    - name: Deploy to GitHub Pages
      id: deployment
      uses: actions/deploy-pages@v4
```
- **When:** Only on `main` branch, only on `push` events
- **Depends:** Waits for `test` job to complete
- **Environment:** Uses `github-pages` environment
- **What:** Deploys the HTML report to GitHub Pages
- **Action:** Official `actions/deploy-pages@v4`
- **URL:** Returns deployment URL in `steps.deployment.outputs.page_url`

## Conditional Execution

### Always runs:
```yaml
if: always()
```
These steps run even if tests fail:
- Parse results
- Job summary
- Unit test publishing
- Artifact upload
- Pages artifact upload

### PR-specific:
```yaml
if: always() && github.event_name == 'pull_request'
```
Only runs on PR events:
- PR comment step

### Main-branch only:
```yaml
if: always() && github.ref == 'refs/heads/main'
```
Only on main branch:
- Pages artifact upload
- Deploy job (combined with push event check)

## Common Customizations

### Change GitHub Pages branch
In `Settings → Pages`, select different branch or custom domain.

### Disable PR comments
Remove or comment out the "Comment on Pull Request" step.

### Disable Pages deployment
Remove the "Upload Pages artifact" step and `deploy-pages` job.

### Customize job summary
Edit the job summary step's markdown template.

### Customize PR comment
Edit the PR comment step's body template.
